# HSP v0.1.1

Release date: 2026-06-01

This is the security-hardening and owner-operated production readiness release.
It keeps the public protocol/API surface stable from `v0.1.0`, but closes the
security scan findings and updates the release evidence for first-party
deployments controlled by the project owner.

## What Changed

- Runtime KMS and edge signing material now fail closed:
  - `HSP_KMS_SEED` is required for local/dev encrypted-store providers.
  - `HSP_EDGE_SIGNING_SECRET` is required for S3/CDN edge-token features.
  - known legacy defaults are rejected.
- AWS KMS runtime mode must be explicit through `HSP_AWS_KMS_RUNTIME`.
- Native wire `DATA` frames enforce bounded payload length before allocation.
- Replay protection is persisted through the encrypted store and survives
  service restart.
- S3/CDN authorization was tightened:
  - bucket/key scope is enforced before distribution operations.
  - direct CID reads require proven namespace binding for scoped credentials.
  - CDN cache keys are tenant-scoped and canonicalized.
  - edge tokens are bound to request method.
- Go toolchain baseline moved to `go1.25.10`.
- `golang.org/x/net` and related indirect dependencies were updated to clear
  reachable `govulncheck` findings.
- CI and release bundle scripts now use the same Go baseline.
- Owner-operated internal security sign-off was added for first-party
  deployments.

## Production Boundary

For first-party owner-operated projects, this release is considered production
ready when deployed with the hardened profile and unique runtime secrets.

For public third-party SaaS exposure, keep the independent external security
review gate from `docs/release-evidence/external-review.md`.

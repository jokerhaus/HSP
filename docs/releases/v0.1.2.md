# HSP v0.1.2

Release date: 2026-06-19

This is a security-fix and release-hardening patch release. It keeps the public
protocol/API surface compatible with `v0.1.1`, but closes validated findings
from the Codex Security scan across HSP core, S3-like storage, CDN edge,
transport body handling, object-lock semantics, SigV4 integrity, CI, and
deployment examples.

## What Changed

- CDN namespace cache hits no longer bypass scoped authorization; namespace
  delivery is revalidated through the shared service layer.
- Native `SUBSCRIBE` now authorizes every requested filter, not only the first.
- `PUT_CHUNK` validates actual ciphertext length against declared length,
  manifest metadata, and chunk limits before storage.
- Upload authorization now enforces `storage_classes` from capability claims.
- Native, gateway, S3-like, and CDN request body reads now have explicit limits.
- Legal hold and retention controls were hardened:
  - ordinary writers cannot clear legal hold.
  - active retention cannot be shortened without `admin.repair`.
  - active compliance retention cannot be shortened or cleared.
  - `CopyObject` cannot overwrite locked destinations.
- Header SigV4 now verifies `x-amz-content-sha256` against the received body.
- S3 path-style routing preserves empty object-key segments.
- The S3 backing-store reader now rejects oversized backend responses.
- Direct `rustls-pemfile` usage was removed.
- Go toolchain baseline moved to `go1.25.11`.
- CI GitHub Actions are pinned to immutable commit SHAs and security tooling is
  pinned to explicit versions.
- Docker base images are digest-pinned and Kubernetes examples use digest-form
  images.

## Verification Used For This Release

- `cargo test --workspace --all-targets`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `env GIT_CONFIG_GLOBAL=/dev/null GIT_CONFIG_SYSTEM=/dev/null cargo audit`
- `GOTOOLCHAIN=go1.25.11+auto go test ./sdk/go/... ./cli/hspctl/...`
- `govulncheck` for `sdk/go`
- `govulncheck` for `cli/hspctl`
- `cargo run -p hsp-conformance`

## Operator Notes

- Public profile remains ciphertext-only.
- `trusted-edge-v1` remains opt-in and is not enabled by default.
- Replace the digest placeholders in `deploy/kubernetes/*.yaml` with the
  actual published image digests before applying manifests.
- Generate release review bundles with a preinstalled, pinned Syft binary.

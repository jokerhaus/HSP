# Upgrade Guide: HSP v0.1.2

This release upgrades `v0.1.1` with security fixes and release-pipeline
hardening. The public HSP, gateway, S3-like, and CDN-like API surfaces remain
compatible.

## Required Operator Changes

### Align Go Toolchain

Use Go `1.25.11` or newer for SDK/CLI validation:

```bash
GOTOOLCHAIN=go1.25.11+auto go work sync
GOTOOLCHAIN=go1.25.11+auto go test ./sdk/go/... ./cli/hspctl/...
```

### Replace Kubernetes Image Digest Placeholders

The Kubernetes manifests now use digest-form image references. The checked-in
digests are placeholders for example manifests. Replace them with the actual
image digests produced by your build and registry before deployment.

### Install Syft Before Release Bundle Generation

The release bundle script no longer downloads and runs the Syft installer. Use
a pinned Syft release installed by your workstation or CI image, then run:

```bash
./scripts/generate_release_review_bundle.sh
```

## Security Behavior Changes

- Namespace CDN responses are re-authorized through the shared service layer
  instead of being served directly from scoped cache hits.
- `SUBSCRIBE` requests with multiple filters require every filter to pass
  namespace/path policy.
- `PUT_CHUNK` rejects mismatches between actual ciphertext length and declared
  chunk length.
- `storage_classes` capability claims are enforced for upload initialization.
- S3/CDN/gateway/native request bodies are bounded before buffering.
- Object lock controls block ordinary writer attempts to clear legal hold,
  shorten retention, or overwrite locked destinations through `CopyObject`.
- Header SigV4 rejects a signed payload hash that does not match the body.
- S3 path-style object keys preserve empty path segments.

## Recommended Validation

```bash
cargo test --workspace --all-targets
cargo clippy --workspace --all-targets -- -D warnings
env GIT_CONFIG_GLOBAL=/dev/null GIT_CONFIG_SYSTEM=/dev/null cargo audit
GOTOOLCHAIN=go1.25.11+auto go test ./sdk/go/... ./cli/hspctl/...
cd sdk/go && GOTOOLCHAIN=go1.25.11+auto "$(go env GOPATH)"/bin/govulncheck ./...
cd ../../cli/hspctl && GOTOOLCHAIN=go1.25.11+auto "$(go env GOPATH)"/bin/govulncheck ./...
cargo run -p hsp-conformance
```

## Rollback Notes

Rollback from `v0.1.2` to `v0.1.1` is protocol-compatible, but not recommended
for production because it reopens the security findings fixed in this release.
If rollback is unavoidable, keep encrypted-store backups and runtime secret
material together and preserve the same KMS domain.

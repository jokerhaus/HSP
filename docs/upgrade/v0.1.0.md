# Upgrade Guide: HSP v0.1.0

This is the first tagged HSP release. There is no previous stable release to
migrate from, so this document focuses on how to adopt `v0.1.0` safely.

## Who Should Read This

Read this if you are integrating an application with HSP as a storage backend,
running HSP services, or testing compatibility through `hsp-s3` or `hsp-cdn`.

## What Changed In v0.1.0

- HSP now has a complete first release surface:
  native runtime, HTTP/3 gateway, S3-like service, CDN-like service, Go SDK,
  CLI, and conformance harness.
- Storage clients can verify existence and integrity without downloading the
  object bytes.
- Operators can scrape Prometheus metrics and inspect structured, secret-aware
  logs.
- Public profile behavior is fixed as ciphertext-only.

## Application Changes

### Use Metadata Checks Instead Of Full Downloads

Applications should prefer metadata checks when they only need to confirm that
an object exists and matches expected integrity.

HTTP gateway by CID:

```http
GET /v1/objects/cid/{cid}/metadata?tenant_id=tenant-alpha
```

HTTP gateway by namespace/path:

```http
GET /v1/objects/namespace/{namespace}/{path}/metadata?tenant_id=tenant-alpha
```

Expected response shape:

```json
{
  "exists": true,
  "deleted": false,
  "size_bytes": 1786060,
  "ciphertext_size_bytes": 1786200,
  "integrity_hash": "sha256-...",
  "cid": "sha256-...",
  "object_cid": "sha256-...",
  "manifest_cid": "sha256-...",
  "created_at_ms": 1776900000000,
  "encryption_profile_id": "public-e2ee-v1",
  "key_policy_id": "policy-default"
}
```

In `v0.1.0`, `integrity_hash`, `cid`, `object_cid`, and `manifest_cid` refer
to the canonical object identity. The object identity is currently the
deterministic manifest CID.

### Keep Uploads Ciphertext-First

Public-profile clients must encrypt object data before upload. HSP stores and
serves ciphertext. If your application needs plaintext at an edge, use a
separate `trusted-edge-v1` policy review and do not enable it by default.

### Handle Integrity Errors

If `PUT_CHUNK` returns `chunk_cid_mismatch`, the client sent bytes that do not
match the declared ciphertext chunk CID. Treat this as a data integrity failure:

- do not retry the same bytes with the same CID
- recompute the chunk CID locally
- restart the upload session if the manifest was built with incorrect chunk
  references

## Operator Changes

### Scrape Metrics

Prometheus metrics are exposed at:

```http
GET /metrics
```

The request must use capability auth with `admin.metrics.read`.

Key metric families:

- `hsp_requests_total`
- `hsp_request_latency_ms_sum`
- `hsp_object_size_bytes_sum`

Labels include operation, tenant, namespace, status code, and outcome.

### Read Structured Logs

Recent structured logs are exposed at:

```http
GET /v1/observability/logs
```

The request must use capability auth with `admin.audit.read`. Logs are
secret-aware and must not include plaintext payloads, raw tokens, or plaintext
keys.

### Run Release Gates

Before deploying this release, run:

```bash
cargo test --workspace --all-targets
cargo clippy --workspace --all-targets -- -D warnings
env GIT_CONFIG_GLOBAL=/dev/null GIT_CONFIG_SYSTEM=/dev/null cargo audit
GOTOOLCHAIN=go1.25.9+auto go test ./sdk/go/... ./cli/hspctl/...
cd sdk/go && GOTOOLCHAIN=go1.25.9+auto "$(go env GOPATH)"/bin/govulncheck ./...
cd ../../cli/hspctl && GOTOOLCHAIN=go1.25.9+auto "$(go env GOPATH)"/bin/govulncheck ./...
cd ../.. && cargo run -p hsp-conformance
```

## Compatibility Notes

- `hsp-s3` is S3-like, not a full AWS S3 clone.
- `hsp-cdn` is HSP-aware edge delivery, not a generic CDN replacement.
- Public-profile `GET` returns ciphertext.
- `GET prefer=raw` remains unsupported in the public profile.
- Object identity remains `object_cid == manifest_cid` in this release.

## Rollback

Because this is the first stable tag, rollback means redeploying the previous
commit you were testing before adopting `v0.1.0`. Keep encrypted store backups
and key material recovery procedures ready before first production deployment.
